# MacBridge

> 让 Windows 用户无痛切换到 Mac 的键盘适配工具

## 一句话介绍

当你从 Windows 切换到 Mac，最痛的不是系统 UI，而是**肌肉记忆对不上**：`Ctrl+C` 变 `Cmd+C`、`Delete` 键方向反了、`Home` 键跳到的位置不一样……MacBridge 做的事情很简单——**让你的 Mac 在你需要时表现得像 Windows**，平滑渡过肌肉记忆的适应期。

## 核心价值

- **零配置上手**：插上外接键盘自动识别 Win/Mac 布局
- **只改最关键的几个键**：不做 Karabiner 那种全量键位定制，只解决 90% 用户每天都撞的痛点
- **可随时一键关闭**：当你肌肉记忆建立后，逐步关闭映射，真正迁移到 Mac 习惯

## 目标用户

- 从 Windows 转到 Mac 的新用户（入职换电脑、买了第一台 Mac）
- 日常同时使用 Win 和 Mac 两台机器的开发者 / 办公人员
- 使用外接 Windows 键盘（如 HHKB、IKBC、Keychron Win 布局）的 Mac 用户

## 非目标（不做什么）

- ❌ 不做完整的键位自定义（那是 Karabiner-Elements 的赛道）
- ❌ 不做宏、不做 tap-hold、不做应用级脚本
- ❌ 不做跨平台同步云配置

> 做一件事并做好：让 Windows 用户在 Mac 上不别扭。
